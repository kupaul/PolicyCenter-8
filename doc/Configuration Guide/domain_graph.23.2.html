PolicyCenter Configuration Guide : Data Model Configuration : The Domain Graph : Domain Graph Overview

Domain Graph Overview

The domain graph is a set of entities. The domain graph defines an aggregate cluster of associated objects that PolicyCenter treats as a single unit for purposes of data changes. Each aggregate cluster has a root and a boundary.
•
The root is a single, specific entity that the aggregate cluster contains. The root entity is the main entity in the graph. A root entity is application-specific.
In PolicyCenter, the root entity is the PolicyPeriod.
•
The boundary defines what is inside the aggregate cluster of objects—all the entities that are part of the graph.
In PolicyCenter, the boundary defines the entities that relate to a PolicyPeriod, such as Job, Note, and effective dated entities such as PolicyAddress or PolicyDriver.
The tables associated with objects in the aggregate cluster, along with their relationships with each other, form the bounded object graph. PolicyCenter constructs the object graph by starting at the root table and collecting all the tables that are owned by the root object or its children, but excluding table views. The end result is a Directed Acyclic Graph (DAG) that starts at the root entity.
