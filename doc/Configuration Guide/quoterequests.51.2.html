Quote Request Handling Overview

To handle high volumes of quote requests, the QuotingProcessor class provides an API for generating quotes. This class calls methods on the Quoting Data (QuotingDataPlugin) plugin, which can save these quotes to a separate database. You can write a custom web service that you publish from PolicyCenter. Your web service receives quote requests from external applications and sends them to the QuotingProcessor API. See “Web Services Introduction” in the Integration Guide.
The following illustration presents a simplified model of how a carrier can use this API and plugin to handle quote requests.
1.
Website A enables potential customers to view policy quotes gathered from multiple insurance carriers. A potential customer on Website A enters basic policy information and requests quotes. One of these insurance carriers uses PolicyCenter to handle these requests.
2.
Website A sends the policy information to multiple carriers. One of these requests goes to a carrier with PolicyCenter.
The carrier has multiple instances of PolicyCenter to handle these requests.
3.
The web service formats the request. The base configuration does not contain an example of this type of web service. The web service verifies the data, then formats the data in the GX model of a PolicyPeriod. See “The Guidewire XML (GX) Modeler” in the Gosu Reference Guide.
4.
The web service then calls the quoteSubmission method in the Quoting Processor API (gw.quoting.QuotingProcessor) to generate a quote.
5.
The quoteSubmission method calls the Quoting Data plugin (QuotingDataPlugin) to obtain an account. Then this method generates a policy period and a quote. This method returns the quote and a quote identifier.
Generating the policy period and quote require the PolicyCenter database. However, in the base implementation, neither the policy period nor the quote are committed to the PolicyCenter database.
6.
The quoteSubmission method calls the Quoting Data plugin to save this quote to the quoting database. The quoting database is partitioned so that it can concurrently handle requests from multiple instances of PolicyCenter.
7.
If the potential customer decides to purchase the carrier’s policy, then the PolicyCenter system of record uses the quote identifier to retrieve the quote from the quoting database. The policy can be bound and saved to the PolicyCenter database.
