Policy Lines Page

Add, delete, and edit policy lines. The Policy Lines table shows each policy line that has been defined:

 

Column

Description

Icon

Icon that represents the policy line.

Sequence

Order in which the policy line appears in PolicyCenter, relative to other policy lines, on the Line Selection screen in a package product.

Name

Name of the policy line. Click the name to open the Policy Line home page where you can edit the details of the selected policy line.

Code

Unique identifier for the policy line.

Other operations

• Sort – Click any column header to sort by that column. Click again to reverse the sort order.

• Change order – If the table is sorted by Sequence, drag a single row to change the order in which it is listed. Alternatively, edit the sequence number to establish a new order. Sequence determines the order in which items appear in PolicyCenter PCF objects when they are sorted by Priority.

• Edit – Click the blue item name to open a page where you can edit the item.

 

 

 

Adding a Policy Line

Adding a policy line creates a instance of an existing policy line pattern that can be independently modified. The new policy line must be based on an existing policy line type. To create a new policy line type, a developer must define a new PolicyLine subtype in the PolicyCenter data model using Studio.

To add a new policy line

1. Click Add.

2. In the Add Policy Line dialog box, fill in the following required fields:

 

Field

Description

Code

Unique identifier for the policy line, which must begin with a letter and be composed only of letters, numbers, and underscores.

Name

Name of the policy line as it appears in PolicyCenter.

Policy Line Type

Type of policy line pattern to add. Select from among all of the types that have been defined in the PolicyCenter data model.

Available Coverage Currency

Name of one currency to add to the policy line. Every policy line must have at least one coverage currency. You can add more coverage currencies later. The set of coverage currencies added to the policy line can be selected from a drop-down list in PolicyCenter when working with this policy line.

3. Click OK to add the new policy line. The resulting Policy Line home page enables you to edit your settings and define all other details of the policy line.

Deleting Policy Lines

1. Navigate to the Product home page for each of the products that use the policy line, and then remove the policy line from the product. You cannot delete a policy line that is used by a product.

2. Navigate back to the Policy Lines page, and then select the policy line to delete by clicking anywhere in its row except the policy line name. To delete multiple policy lines at once, use standard keyboard and mouse key combinations to select multiple rows. The selected rows are highlighted.

3. Click Delete.

The deleted policy line is added to your active change list. It is not deleted from the PolicyCenter configuration until you commit your changes.

Common Operations

To revert changes to policy lines

 

The changes you make in the current page are stored in your active change list. You can

• Cancel all changes you have made in the current page by clicking Revert.

• Selectively cancel changes by clicking Changes to display the Changes panel, then clicking to open the Changes page, then clicking Revert in the Revert column.

Note:  When you click Revert, Product Designer reloads the entire product model from the workspace.
