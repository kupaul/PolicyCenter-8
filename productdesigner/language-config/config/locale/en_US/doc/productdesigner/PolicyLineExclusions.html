Exclusions Page

Manage exclusions.

 

Column

Description

Sequence

Order in which the exclusion appears in PolicyCenter, relative to other exclusions.

Name

Exclusion name. Click to open the Exclusion home page where you can edit the details of the selected exclusion.

Code

Unique identifier for the exclusion.

Category

Coverage category assigned to this exclusion. Coverage categories permit grouping and searching in PolicyCenter so you can more easily find specific exclusions.

Covered Object

Coverable to which this exclusion applies. Coverables are defined in Studio as part of the data model.

Other operations

• Sort – Click any column header to sort by that column. Click again to reverse the sort order.

• Change order – If the table is sorted by Sequence, drag a single row to change the order in which it is listed. Alternatively, edit the sequence number to establish a new order. Sequence determines the order in which items appear in PolicyCenter PCF objects when they are sorted by Priority.

• Edit – Click the blue item name to open a page where you can edit the item.

 

 

 

Adding an Exclusion

1. Click Add.

2. In the Add Exclusion dialog box, fill in the following required fields:

 

Field

Description

Code

Unique identifier for the exclusion, which must begin with a letter and be composed only of letters, numbers, and underscores.

Name

Name of the exclusion as it appears in PolicyCenter.

Category

Coverage category to assign to this exclusion. Coverage categories permit grouping and searching in PolicyCenter so you can more easily find specific exclusions. The category you select does not affect the exclusion in any other way. To define new coverage categories go to the Categories page.

Covered Object

Coverable to which this exclusion applies. Coverables are defined in Studio as part of the data model.

Covered Object Type

Coverage table in which instances of this exclusion are stored. Types are defined in Studio as part of the data model.

3. Click OK to add the new exclusion. The resulting Exclusion home page enables you to edit your settings and define additional exclusion details.

Common Operations

To revert changes to exclusions

 

The changes you make in the current page are stored in your active change list. You can

• Cancel all changes you have made in the current page by clicking Revert.

• Selectively cancel changes by clicking Changes to display the Changes panel, then clicking to open the Changes page, then clicking Revert in the Revert column.

Note:  When you click Revert, Product Designer reloads the entire product model from the workspace.

 

 

 

To delete exclusions

 

1. Select the row to delete by clicking anywhere in the row. To delete multiple items at once, select multiple rows.

2. Click Delete.

Deleted values are added to your active change list. They are not deleted from the PolicyCenter configuration until you commit your changes.

 

 

To navigate to related pages

Use the links under Go to to display pages where you can specify additional policy line properties. The available links vary depending on the data model of the line of business.
