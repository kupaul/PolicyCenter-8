Schedule Details Section Settings

 

Field

Description

Round to Next Month on Day

Appears only when Audit Type is Premium Report and Scheduled is Monthly.

Calendar day of the month on which to switch to the following month. Enables carriers to accommodate a first premium report occurring in a partial month. When rounding occurs, the first report covers the first short month and the second month. Subsequent premium reports correspond to calendar months.

Schedule Basis

Appears only when Audit Type is Premium Report.

Basis for scheduling premium reports. One of:

• Calendar month

• Policy month

Minimum Audit Period (Days)

Appears only when Audit Type is Premium Report.

Minimum number of days in an audit period. Generally affects premium reports that occur at the end of a policy period when the premium report at the beginning of the policy period was rounded.

Exclude Last Audit Period

Appears only when Audit Type is Premium Report.

Whether to exclude the last audit period. Enables carriers to substitute a final audit in place of the last premium report.

Single Checking Ends After Policy Period Start (days)

Appears only when Audit Type is Checking Audit.

Number of days after the beginning of the policy period to conduct a single checking audit.

Type of Days

Appears only when Audit Type is Checking Audit.

Whether to count the days as business days or calendar days.

Editing Available Coverage Currencies

You selected one coverage currency when you added the policy line. You now can add more coverage currencies that users can select while working with the line in PolicyCenter. Click Expand to expand the Available Coverage Currencies section.

To add an available coverage currency

1. Under Available Coverage Currencies, click Add and then select from the list of coverage currencies that appears.

2. Repeat to add more available coverage currencies. Note that if all coverage currencies have been added, the Add command is unavailable.

Note:  Additional coverage currencies can be defined in Studio.

To remove an available coverage currency

1. Under Available Coverage Currencies, click the name of the currency to remove.

2. Click Remove.

Every policy line must have at least one coverage currency defined. Therefore, you cannot remove the last available coverage currency.

Common Operations

To cancel addition or revert changes to audit schedule settings

 

The changes you make in the current page are stored in your active change list. You can

• Cancel all changes you have made in the current page by clicking Revert.

• Selectively cancel changes by clicking Changes to display the Changes panel, then clicking to open the Changes page, then clicking Revert in the Revert column.

Note:  When you click Revert, Product Designer reloads the entire product model from the workspace.

 

 

 

To validate changes to audit schedule settings

 

Validate your changes by clicking Validate. Validation errors appear at the top of the page on a red background. The errors are marked with and appear on a red background. When no validation errors exist, clicking Validate displays the message No validation errors marked with on a green background.

Note:  When you click Validate, Product Designer reloads the entire product model from the workspace. Therefore, Product Designer can display unexpected values after validation if other users have committed changes to the same workspace.

 

 

To delete the audit schedule

• Click Delete.

• If you have not yet committed the new audit schedule, Product Designer deletes the audit schedule.

• If the audit schedule has been committed, the portion of the page that was previously editable becomes gray and you cannot make further changes to it.

 

The deleted item is added to your active change list. It is not deleted from the PolicyCenter configuration until you commit your changes.

Prior to committing your change list, you can click Cancel Deletion to restore the deleted item. The gray overlay is removed and you can once again edit the item.
