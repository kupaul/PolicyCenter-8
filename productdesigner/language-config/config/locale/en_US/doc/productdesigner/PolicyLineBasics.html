Editing Basic Policy Line Settings

 

Name

Name of the policy line, initially set in the Add Policy Line dialog box. You can:

 

• Edit the text directly in the field using the current language. You can set the current language in the User Settings page.

• Click Translate to display the Display Key Values by Language dialog box, where you can enter or edit the text in any supported language.

Editing Available Coverage Currencies

You selected one coverage currency when you added the policy line. You now can add more coverage currencies that users can select while working with the line in PolicyCenter. Click Expand to expand the Available Coverage Currencies section.

To add an available coverage currency

1. Under Available Coverage Currencies, click Add and then select from the list of coverage currencies that appears.

2. Repeat to add more available coverage currencies. Note that if all coverage currencies have been added, the Add command is unavailable.

Note:  Additional coverage currencies can be defined in Studio.

To remove an available coverage currency

1. Under Available Coverage Currencies, click the name of the currency to remove.

2. Click Remove.

Every policy line must have at least one coverage currency defined. Therefore, you cannot remove the last available coverage currency.

Editing Advanced Policy Line Settings

Click Expand to expand the Advanced section.

 

Field

Description

Policy Line Type

Policy line type that you selected when you added the policy line. Established when you add a new policy line and thereafter cannot be changed.

Territory Code Needed

Whether the policy line needs to use territory codes. Used for rating purposes. If enabled, one and only one territory code must be entered for each location in this policy line.

Scripts     Initialization

Gosu script that runs when the PolicyLinePattern creates a new Policy. The script has a PolicyLine symbol available that PolicyCenter correctly types based on the configuration information of the PolicyLinePattern. Use this script to perform actions when initializing the PolicyLine. In the base configuration, Businessowners and General Liability lines include sample initialization scripts that initialize certain coverage terms.

Common Operations

To cancel addition or revert changes to basic policy line settings

 

The changes you make in the current page are stored in your active change list. You can

• Cancel all changes you have made in the current page by clicking Revert.

• Selectively cancel changes by clicking Changes to display the Changes panel, then clicking to open the Changes page, then clicking Revert in the Revert column.

Note:  When you click Revert, Product Designer reloads the entire product model from the workspace.

 

 

 

 

To validate changes to basic policy line settings

Validate your changes by clicking Validate. Validation errors appear at the top of the page on a red background. The errors are marked with and appear on a red background. When no validation errors exist, clicking Validate displays the message No validation errors marked with on a green background.

Note:  When you click Validate, Product Designer reloads the entire product model from the workspace. Therefore, Product Designer can display unexpected values after validation if other users have committed changes to the same workspace.

 

 

To delete the policy line

1. Navigate to the Product home page for each of the products that use the policy line, and then remove the policy line from the product. You cannot delete a policy line that is used by a product.

2. Navigate back to the home page for the policy line you were previously viewing.

3. Click Delete.

• If you have not yet committed the new policy line definition, Product Designer deletes the policy line.

• If the policy line definition has been committed, the portion of the page that was previously editable becomes gray and you cannot make further changes to it.

The deleted item is added to your active change list. It is not deleted from the PolicyCenter configuration until you commit your changes.

Prior to committing your change list, you can click Cancel Deletion to restore the deleted item. The gray overlay is removed and you can once again edit the item.

 

 

To navigate to related pages

Use the links under Go to to display pages where you can specify additional policy line properties. The available links vary depending on the data model of the line of business.
