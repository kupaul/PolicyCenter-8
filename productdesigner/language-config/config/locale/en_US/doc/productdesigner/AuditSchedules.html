Audit Schedules Page

Add, delete, and edit audit schedules. The Audit Schedules table shows each audit schedule that has been defined:

 

Column

Description

Name

Name of the audit schedule. Click the name to open the Audit Schedule home page where you can edit the details of the selected audit schedule.

Code

Unique identifier for the audit schedule.

Other operations

• Sort – Click any column header to sort by that column. Click again to reverse the sort order.

• Edit – Click the name to open a page where you can edit the item.

 

 

Adding an Audit Schedule

Add a new audit schedule if the existing audit schedules do not meet your needs.

Note:  In the default configuration, all audit schedules are shared by all lines of business that use audit schedules.

To add a new audit schedule

1. Click Add.

2. In the Add Audit Schedule dialog box, fill in the following required fields:

 

Field

Description

Code

Unique identifier for the audit schedule, which must begin with a letter and be composed only of letters, numbers, and underscores.

Name

Name of the audit schedule as it appears in PolicyCenter.

Audit Type

Type of audit schedule, one of:

Checking Audit – Performed on installment plans to verify the installment amount. Does not generate credits or debits to the account.

Premium Report – Periodic audit of basis amounts to determine if the current premium is correct. Can result in credits or debits to the account.

Final Audit – Final assessment of basis amounts that results in a credit or debit to the account. Audits that occur on a policy cancellation or expiration are final audits.

3. Click OK to add the new audit schedule. The resulting Audit Schedule home page enables you to edit your settings and define all other details of the audit schedule.

Common Operations

To revert changes to audit schedules

 

The changes you make in the current page are stored in your active change list. You can

• Cancel all changes you have made in the current page by clicking Revert.

• Selectively cancel changes by clicking Changes to display the Changes panel, then clicking to open the Changes page, then clicking Revert in the Revert column.

Note:  When you click Revert, Product Designer reloads the entire product model from the workspace.

 

To delete audit schedules

1. Select the row to delete by clicking anywhere in the row. To delete multiple items at once, select multiple rows.

2. Click Delete.

Deleted values are added to your active change list. They are not deleted from the PolicyCenter configuration until you commit your changes.
