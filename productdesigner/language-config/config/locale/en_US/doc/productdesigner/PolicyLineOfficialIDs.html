Official IDs Page

Configure official IDs for a line of business. Official IDs are identification codes needed by a policy line, such as tax IDs, bureau IDs, and state IDs.

In the base configuration, only the Workers’ Compensation line uses official IDs. Adding official IDs to other lines requires the following additional steps to display the appropriate fields in PolicyCenter:

1. Open the property file for the policy line pattern in the PolicyCenter installation directory:

resources/configuration/config/resources/productmodel/policylinepatterns/properties

The properties file is named code.properties where code is the policy line code. For example, the workers’ compensation properties file is WorkersCompLine.properties.

2. Set the line.usesOfficialIDs property to true to add the fields, or false to remove the fields.

For more information about enabling official IDs, see “Adding the Optional Tabs and Fields to Policy Lines” in the Product Model Guide.

 

Column

Description

Official ID type

Type of ID, selected from the Official ID Type typelist.

Applicable States

Comma-separated list of states to which the ID applies, selected from the Applicable States typelist.

Scope

The scope of the official ID, one of the following selected from the Official ID Scope typelist:

• State – Official ID applies to the primary named insured only. Generally, you must specify a different ID for each state in which this ID applies. The Bureau ID in workers’ compensation is an example of an ID that varies by state.

• Insured and State – Official ID applies to additional named insureds. The base configuration of workers’ compensation captures the tax ID of each additional named insured as an official ID on the policy.

Interstate

Whether the official ID applies to multiple states. In the base configuration, Interstate is true for NCCI states, where one official ID applies to a collection of states.

Common Operations

To revert changes to official IDs

 

The changes you make in the current page are stored in your active change list. You can

• Cancel all changes you have made in the current page by clicking Revert.

• Selectively cancel changes by clicking Changes to display the Changes panel, then clicking to open the Changes page, then clicking Revert in the Revert column.

Note:  When you click Revert, Product Designer reloads the entire product model from the workspace.

 

 

 

To validate official IDs

Validate your changes by clicking Validate. Validation errors appear at the top of the page on a red background. The errors are marked with and appear on a red background. When no validation errors exist, clicking Validate displays the message No validation errors marked with on a green background.

Note:  When you click Validate, Product Designer reloads the entire product model from the workspace. Therefore, Product Designer can display unexpected values after validation if other users have committed changes to the same workspace.

 

 

To navigate to related pages

Use the links under Go to to display pages where you can specify additional policy line properties. The available links vary depending on the data model of the line of business.
