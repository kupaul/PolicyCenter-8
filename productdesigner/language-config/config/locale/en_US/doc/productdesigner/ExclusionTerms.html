Coverage Terms for Exclusions Page

Configure coverage terms for exclusions.

 

Column

Description

Sequence

Order in which the term appears in PolicyCenter, relative to other terms.

Name

Term name. Click to open the Coverage Term home page where you can edit the details of the selected term.

Code

Unique identifier for the term.

Model Type

How the coverage term value measures the coverage and how the rating engine uses the value. In the base configuration, every coverage term has one of the following model type values:

• Coinsurance

• Deductible

• Exclusion

• Limit

• Peril

• Other – Information about the coverage that is not a deductible, exclusion, limit, or coinsurance.

Other operations

• Sort – Click any column header to sort by that column. Click again to reverse the sort order.

• Change order – If the table is sorted by Sequence, drag a single row to change the order in which it is listed. Alternatively, edit the sequence number to establish a new order. Sequence determines the order in which items appear in PolicyCenter PCF objects when they are sorted by Priority.

• Edit – Click the blue item name to open a page where you can edit the item.

 

 

 

Adding a Coverage Term for an Exclusion

1. Click Add.

2. In the Add Term dialog box, fill in the following fields:

 

Field

Description

Code

Unique identifier for the coverage term, which must begin with a letter and be composed only of letters, numbers, and underscores. Established when you add a new term and thereafter cannot be changed.

Name

Name of the coverage term as it appears in PolicyCenter.

Term Type

Type of coverage term, one of:

• Direct – Numeric value entered directly by the PolicyCenter user, subject to the bounds specified in the CoverageTermPattern.

• Option – Numeric value selected from a predefined range of term options. Define the list of options in the Coverage Term Options page

• Package – Compound collection of limits or deductibles selected as a group from a list of predefined choices.

• Typekey – Value selected from a typelist of predefined choice, optionally filtered through use of a type filter.

• Generic – Custom value, for example, a date or Boolean value.

Typelist

Appears only if the Term Type is Typekey.

Typelist from in which the term type is defined.

Required

Whether the exclusion requires this coverage term. If selected, PolicyCenter displays an error if you do you supply a value for the coverage term.

Column

Database table column to use for the coverage term.

3. Click OK to add the new coverage term. The resulting Coverage Term home page enables you to edit your settings and define additional coverage term details.

Common Operations

To revert changes to coverage terms on exclusions

 

The changes you make in the current page are stored in your active change list. You can

• Cancel all changes you have made in the current page by clicking Revert.

• Selectively cancel changes by clicking Changes to display the Changes panel, then clicking to open the Changes page, then clicking Revert in the Revert column.

Note:  When you click Revert, Product Designer reloads the entire product model from the workspace.

 

 

 

To delete coverage terms on exclusions

 

1. Select the row to delete by clicking anywhere in the row. To delete multiple items at once, select multiple rows.

2. Click Delete.

Deleted values are added to your active change list. They are not deleted from the PolicyCenter configuration until you commit your changes.

 

 

To navigate to related pages

Use the links under Go to to display pages where you can specify additional exclusion properties.
