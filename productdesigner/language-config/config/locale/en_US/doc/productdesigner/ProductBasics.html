Editing Basic Product Settings

 

Name

Name of the product, initially set in the Add Product dialog box. You can:

 

• Edit the text directly in the field using the current language. You can set the current language in the User Settings page.

• Click Translate to display the Display Key Values by Language dialog box, where you can enter or edit the text in any supported language.

 

 

Description

Internal product description that can be viewed by other business analysts. You can

 

• Edit the text directly in the field using the current language. You can set the current language in the User Settings page.

• Click Translate to display the Display Key Values by Language dialog box, where you can enter or edit the text in any supported language.

 

 

Abbreviation

Product abbreviation that Guidewire displays in certain fields where the full product name does not fit, such as in the Submission screen. For example, the abbreviation for the Personal Auto product is PA. In the Submission screen, the title bar displays PA Submission.

Default Policy Term

Default contractual term of a policy within this product, initially set in the Add Product dialog box.

Product Type

Type of product, Commercial or Personal, initially set in the Add Product dialog box. This value enables PolicyCenter to show or hide portions of certain built-in user interface objects that are designed either for personal or commercial contexts.

Account Type

Types of accounts that can be associated with this product during a submission: Person, Company, or Any. This value initially is set in the Add Product dialog box.

Offering Required

Whether an offering must be selected to enable this product to produce a submission.

Managing Policy Lines

Click Expand to expand the Policy Lines section where you can view, add, remove, and edit policy lines.

To add policy lines

1. Under Policy Lines, click Add and then select from among the list of policy lines that appears.

2. Repeat to add more policy lines. Note that if all policy lines have been added, the Add command is unavailable.

To remove policy lines

1. Under Policy Lines, select a policy line to remove by clicking anywhere in its row except the name. To remove multiple policy lines at once, select multiple rows.

2. Click Remove.

To edit a policy line

1. Under Policy Lines, click the policy line name to jump directly to the policy line’s home page, where you can edit the policy line.

2. Edit the policy line as needed.

3. Click your browser’s Back button to return to product page you were previously viewing.

Adding and Removing Policy Terms

In addition to a default term, a policy can have other terms that can be selected during a submission. Click Expand to expand the Policy Terms section.

To add a policy term

1. Under Policy Term, click Add and then select from the list of policy terms that appears.

2. Repeat to add more policy terms. Note that if all policy terms have been added, the Add command is unavailable.

Note:  Other policy terms can be defined in Studio in the TermType typelist.

Editing Advanced Product Settings

Click Expand to expand the Advanced section.

 

Field

Description

Quote Rounding Level

Default rounding level that PolicyCenter uses in calculating transactions if no other rounding is specified in the rating engine. PolicyCenter rounds transaction amounts to 10‑RoundingLevel. Thus, if RoundingLevel is 2, it rounds to hundredths, or pennies when working with U.S. dollars. If the RoundingLevel is 3, it rounds to thousandths.

Quote Rounding Mode

Default rounding mode for transactions and rating if no other rounding mode is specified in the rating engine. Values are:

• Ceiling – Round up.

• Down – Round down.

• Floor – Round down.

• Half Down – Round to the nearest. If exact halfway, round down. For example, round 2.5000 to 2.0000.

• Half Even – Round to the nearest. If exactly halfway, round toward the nearest even number. For example, round 2.5000 to 2.0000, but round 3.5000 to 4.0000.

• Half Up – Round to the nearest. If exactly halfway, round up. For example, round 2.5000 to 3.0000.

• Unnecessary – No rounding is necessary because the value returned is exact.

• Up – Round up.

If you have integrated your own rating engine, configure the product model to match the mode of your rating engine. Then the transaction calculator will use the same rounding mode as your rating engine.

Days Until Quote Needed

Targeted number of days between a submission and delivery of a quote. This value is not used in any of the base PolicyCenter products, but could be configured to produce activities or messages after the specified number of days has elapsed.

Scripts     Initialization

Initialization scripts that PolicyCenter executes every time a new policy is created from the product. Policy creation in this context means starting a submittal job and instantiating the product, not the binding and issuing that completes the legal creation of a policy. In the base configuration, Businessowners, Commercial Auto, and Personal Auto products include sample initialization scripts that set the deposit collected amount to zero. Additional sample initialization scripts are available for use in these products, but are commented out by a “//” prefix. Delete the comment characters to enable the scripts.

Document Templates

Document templates associated with the product.

Managing Document Templates

Document templates specify the documents that can be created by users when working with the product. By default, document templates reside in the file system of the PolicyCenter application server. Through integration, documents typically are produced by a third-party document management system.

Use the Add and Remove links under Document Templates to add and remove templates. This topic explains how to add an existing document template to a product. Creating a new document template is beyond the scope of this topic. For detailed information about creating new document templates, see “Document Production” in the PolicyCenter Integration Guide.

The following steps assume you have already created a document template or that you are using an existing document template provided in the PolicyCenter base configuration.

To add a document template to the product

1. If it is not already in the required location, copy your document template file and document template descriptor file to the required location. In the base configuration, the required location for these files is in modules/configuration/config/resources/doctemplates in the PolicyCenter installation directory.

2. Under Document Templates, click Add to add a new row to the Document Templates table. Note that the new row is added to the end of the list. If needed, scroll down to see the new row. Click a cell in the row to add or edit a value.

3. Enter a type name in the Document Type cell of the new row, or click Search to display the Document Type (DocumentTemplateType) dialog box and select a type.

4. In the Template File cell of the new row, enter the name of the template file to associate with this document template type.

Common Operations

To cancel addition or revert changes to basic product settings

 

The changes you make in the current page are stored in your active change list. You can

• Cancel all changes you have made in the current page by clicking Revert.

• Selectively cancel changes by clicking Changes to display the Changes panel, then clicking to open the Changes page, then clicking Revert in the Revert column.

Note:  When you click Revert, Product Designer reloads the entire product model from the workspace.

 

 

 

 

To validate changes to basic product settings

Validate your changes by clicking Validate. Validation errors appear at the top of the page on a red background. The errors are marked with and appear on a red background. When no validation errors exist, clicking Validate displays the message No validation errors marked with on a green background.

Note:  When you click Validate, Product Designer reloads the entire product model from the workspace. Therefore, Product Designer can display unexpected values after validation if other users have committed changes to the same workspace.

 

 

To delete the product

While editing the product definition, click Delete.

• If you have not yet committed the new product definition, Product Designer deletes the product.

• If the product definition has been committed, the portion of the page that was previously editable becomes gray and you cannot make further changes to it.

The deleted item is added to your active change list. It is not deleted from the PolicyCenter configuration until you commit your changes.

Prior to committing your change list, you can click Cancel Deletion to restore the deleted item. The gray overlay is removed and you can once again edit the item.

 

 

Deleting a product deletes the entire line of business, and can have a significant impact on other product model and data model objects throughout PolicyCenter.

WARNING  Deleting a product can have unanticipated and unintended consequences. Guidewire strongly recommends that you fully understand your product model configuration before taking this action. For example, if you do not correctly remove references to a deleted policy line from your product definition, PolicyCenter cannot synchronize the product model at server startup. As a consequence, it can be impossible to start the server thereafter, until the problem is corrected.

 

To navigate to related pages

Use the links under Go to to display pages where you can specify additional product properties.
