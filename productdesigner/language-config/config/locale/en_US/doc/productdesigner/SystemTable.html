System Table Home Page

Examine and edit the details of the selected system table. The page title includes the system table name followed by the system table code in brackets. For example:

MVRConfig [motor_vehicle_record_configs.xml]

Editing System Tables

The columns available in the selected system table are defined in Studio by a developer. You cannot add or remove system table columns in Product Designer.

You can edit values in place in the system table cells. When selected, each cell has a behavior that corresponds to its type. For example, date cells display a Calendar button that when clicked displays a calendar from which a date can be selected. Certain cells containing string values display a Translate button that when clicked displays the Display Key Values by Language dialog box where you can enter localized values.

Alternatively, you can click View Details to edit values from the selected row in the Row In System Table page.

To add a new row to a system table

1. Click Add.

2. In the Add Row To System Table dialog box, fill in the fields that appear as needed. The fields that appear in this dialog box are those that have been defined in the backing entity for the system table.

3. Click OK to add the new row.

Common Operations

To revert changes to the system table

 

The changes you make in the current page are stored in your active change list. You can

• Cancel all changes you have made in the current page by clicking Revert.

• Selectively cancel changes by clicking Changes to display the Changes panel, then clicking to open the Changes page, then clicking Revert in the Revert column.

Note:  When you click Revert, Product Designer reloads the entire product model from the workspace.

 

 

 

 

To validate the system table

Validate your changes by clicking Validate. Validation errors appear at the top of the page on a red background. The errors are marked with and appear on a red background. When no validation errors exist, clicking Validate displays the message No validation errors marked with on a green background.

Note:  When you click Validate, Product Designer reloads the entire product model from the workspace. Therefore, Product Designer can display unexpected values after validation if other users have committed changes to the same workspace.

 

 

To delete system table rows

 

1. Select the row to delete by clicking anywhere in the row. To delete multiple items at once, select multiple rows.

2. Click Delete.

Deleted values are added to your active change list. They are not deleted from the PolicyCenter configuration until you commit your changes.
