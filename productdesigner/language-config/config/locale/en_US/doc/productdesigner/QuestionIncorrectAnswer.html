Question Incorrect Answer Page

Define PolicyCenter behavior for incorrect answers.

 

Handle Incorrect Answer

Whether PolicyCenter processes incorrect answers. Select this check box to enable the other fields on this page.

Correct Answer

Answer that does not invoke any of the other behaviors defined on this page. The format of this field matches the question type.

Failure Message

Message to display in PolicyCenter when the answer does not match the Correct Answer value. You can:

 

• Edit the text directly in the field using the current language. You can set the current language in the User Settings page.

• Click Translate to display the Display Key Values by Language dialog box, where you can enter or edit the text in any supported language.

 

 

Blocking Action

One of:

• blank – Regardless of answer specified, user can continue through the wizard with no immediate consequence.

• Warn User – If incorrect answer is specified, PolicyCenter displays the Failure Message but does not prevent progress through the wizard.

• Block User – If incorrect answer is specified, PolicyCenter displays the Failure Message and prevents further progress through the wizard until the correct answer is specified. PolicyCenter tracks changes to incorrect answers through job history events.

Note: These selections are defined in the BlockingAction typelist. A developer can use Studio to define additional blocking actions. See the Product Model Guide for more information.

Create UW Issue Type

Underwriting issue, if any, to create when the incorrect answer is supplied. Underwriting issues types are defined in the UWIssueType system table. Examine the CheckingSet column of this system table to understand how various types of underwriting issues affect the outcome of the incorrect answer.

Risk Points

Number of risk points to add when an incorrect answer is specified. A developer can use Studio to establish the number of risk points that trigger various actions. Risk points are evaluated in the Evaluation Æ Segmentation rules. For example, an incorrect answer to this question can be configured to add 50 risk points. When the number of risk points from all questions of the selected UWIssueType is greater than 200, PolicyCenter can raise the underwriting issue and take the appropriate action.

Common operations

To revert changes to the incorrect answer settings

 

The changes you make in the current page are stored in your active change list. You can

• Cancel all changes you have made in the current page by clicking Revert.

• Selectively cancel changes by clicking Changes to display the Changes panel, then clicking to open the Changes page, then clicking Revert in the Revert column.

Note:  When you click Revert, Product Designer reloads the entire product model from the workspace.

 

 

 

To validate the incorrect answer settings

Validate your changes by clicking Validate. Validation errors appear at the top of the page on a red background. The errors are marked with and appear on a red background. When no validation errors exist, clicking Validate displays the message No validation errors marked with on a green background.

Note:  When you click Validate, Product Designer reloads the entire product model from the workspace. Therefore, Product Designer can display unexpected values after validation if other users have committed changes to the same workspace.

 

 

 

To navigate to related pages

Use the links under Go to to display pages where you can specify additional question properties. The available links vary depending on the question type.
