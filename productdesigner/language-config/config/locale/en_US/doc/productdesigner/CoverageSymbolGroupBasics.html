Adding Coverage Symbol Patterns to a Coverage Symbol Group

Click Expand to expand the Coverage Symbol Patterns section.

To add a coverage symbol pattern

1. Under Coverage Symbol Patterns, click Add and then select from the list of coverage symbol patterns that appears.

2. Repeat to add more available coverage symbol patterns. Note that if all coverage symbol patterns have been added, the Add command is unavailable.

Note:  Additional coverage symbol patterns can be defined in Studio.

To remove a coverage symbol pattern

1. Under Available Coverage Patterns, click the name of the pattern to remove.

2. Click Remove.

Every coverage symbol group must have at least one coverage symbol pattern. Therefore, you cannot remove the last available coverage symbol pattern.

Common Operations

To cancel addition or revert changes to coverage symbol group settings

 

The changes you make in the current page are stored in your active change list. If you have just added an item, click Cancel Addition to remove the newly-added item. If you are editing an existing item, cancel all changes you have made in the current page by clicking Revert. Or, selectively cancel changes by clicking Changes to display the Changes panel, then click Show Full Panel to open the Changes page.

 

 

 

To validate the coverage symbol group settings

Validate your changes by clicking Validate. Validation errors appear at the top of the page on a red background. The errors are marked with and appear on a red background. When no validation errors exist, clicking Validate displays the message No validation errors marked with on a green background.

Note:  When you click Validate, Product Designer reloads the entire product model from the workspace. Therefore, Product Designer can display unexpected values after validation if other users have committed changes to the same workspace.

 

 

To delete the coverage symbol group

1. Navigate to the Coverage home page for each of the coverages that use the coverage symbol group, and then remove the coverage symbol group from the coverage. You cannot delete a coverage symbol group that is used by a coverage.

2. Navigate back to the home page for the coverage symbol group you were previously viewing.

3. Click Delete.

• If you have not yet committed the new coverage symbol group definition, Product Designer deletes the group.

• If the coverage symbol group has been committed, the portion of the page that was previously editable becomes gray and you cannot make further changes to it.

The deleted item is added to your active change list. It is not deleted from the PolicyCenter configuration until you commit your changes.

Prior to committing your change list, you can click Cancel Deletion to restore the deleted item. The gray overlay is removed and you can once again edit the item.
