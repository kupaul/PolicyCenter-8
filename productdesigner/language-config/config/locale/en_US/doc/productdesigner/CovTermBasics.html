Coverage Term Home Page

Examine and edit the details of the selected coverage term. The page title includes the coverage term name followed by the coverage term code in brackets. For example:

Collision Deductible [PACollDeductible]

Editing Basic Coverage Term Settings

 

Term Type

Term type of the term as set in the Add Term dialog box. Established when you add a new term and cannot be changed.

Column

Database column that holds the data for the coverage term, as set in the Add Term dialog box. Established when you add a new term and cannot be changed.

Name

Name of the coverage term, initially set in the Add Term dialog box. You can:

 

• Edit the text directly in the field using the current language. You can set the current language in the User Settings page.

• Click Translate to display the Display Key Values by Language dialog box, where you can enter or edit the text in any supported language.

 

 

Description

Description of the coverage term that can be viewed by other business analysts.

 

• Edit the text directly in the field using the current language. You can set the current language in the User Settings page.

• Click Translate to display the Display Key Values by Language dialog box, where you can enter or edit the text in any supported language.

 

 

Required

Whether the coverage requires this coverage term, initially set in the Add Term dialog box. You can change this setting at any time. If selected, PolicyCenter displays an error if you do you supply a value for the coverage term.

Value Type

Type of value this coverage term holds, which enables code that accesses the value to handle the value appropriately.

• Count

• Days

• Hours

• Money

• Percent

Default Value

Appears only if the Value Type is not Money.

Value to prefill in this coverage term when it appears in PolicyCenter.

• If Value Type is Money, you can specify a different default value for each currency the coverage supports.

• If Term Type is Direct, you can specify a minimum and maximum value.

Typelist

Appears only if the Term Type is Typekey.

Typelist from in which the term type is defined, initially set in the Add Term dialog box. You can change the typelist at any time.

Type Filter

Appears only if the Term Type is Typekey.

Optional filter that limits the values to a subset of the typecodes available in the typelist.

Editing Coverage Term Currencies

For coverage terms of type Money, click Add to add a new row to the currency table.

Note:  It is possible to add the same currency multiple times, but doing so results in a validation error.

 

Currency

Currency to allow users to select when defining an instance of this coverage term. Although Product Designer does not enforce this rule, Guidewire recommends that you restrict the choices of currency to the same set specified at the policy line level.

Default Value

Value that appears in this field when this coverage term appears in PolicyCenter.

• If Term Type is Direct, type a value in the cell.

• If Term Type is Option, Package, or Type Key, select a value from the drop down list that appears when you click in the cell.

Minimum Value

Appears only if Term Type is Direct.

Minimum value the user can enter for this coverage term.

Maximum Value

Appears only if Term Type is Direct.

Maximum value the user can enter for this coverage term.

Note:  Additional currencies can be defined in Studio.

To delete a coverage term currency

 

1. Select the row to delete by clicking anywhere in the row. To delete multiple items at once, select multiple rows.

2. Click Delete.

Deleted values are added to your active change list. They are not deleted from the PolicyCenter configuration until you commit your changes.

 

 

Configuring Integration

Click Expand to expand the Integration section.

 

Field

Description

Model Type

How the coverage term value measures the coverage and how the rating engine uses the value. In the base configuration, every coverage term has one of the following model type values:

• Coinsurance

• Deductible

• Exclusion

• Limit

• Peril

• Other – Information about the coverage that is not a deductible, exclusion, limit, or coinsurance.

Aggregation Model

Appears only if the Term Type is Direct or Option.

How the coverage term is aggregated:

• Annual

• Each accident

• Each incident

• Each common cause

• Per claim

• Per item

• Per occurrence

• Per person

Values in this list are defined in the CovTermModelAgg typelist.

Restriction Model

Appears only if the Term Type is Direct or Option.

How the coverage term is restricted, as defined in the CovTermModelRest typelist. A claim system can use this field. Assuming ClaimCenter integration, this value appears in the ClaimCenter Applicable To field on each instance of the this coverage term in the policy data for a claim.

Typical values include Accident, Bodily Injury, Employee dishonesty, Medical, and many others.

Reference Code

Public ID of the modifier as used in a legacy policy system of record.

Common Operations

To cancel addition or revert changes to coverage term settings

 

The changes you make in the current page are stored in your active change list. If you have just added an item, click Cancel Addition to remove the newly-added item. If you are editing an existing item, cancel all changes you have made in the current page by clicking Revert. Or, selectively cancel changes by clicking Changes to display the Changes panel, then click Show Full Panel to open the Changes page.

 

 

 

To validate the coverage term

Validate your changes by clicking Validate. Validation errors appear at the top of the page on a red background. The errors are marked with and appear on a red background. When no validation errors exist, clicking Validate displays the message No validation errors marked with on a green background.

Note:  When you click Validate, Product Designer reloads the entire product model from the workspace. Therefore, Product Designer can display unexpected values after validation if other users have committed changes to the same workspace.

 

 

To delete the coverage term

While editing coverage term settings in the Coverage Term home page, click Delete.

• If you have not yet committed the coverage term, Product Designer deletes the coverage term.

• If the coverage term has been committed, the portion of the page that was previously editable becomes gray and you cannot make further changes to it.

The deleted item is added to your active change list. It is not deleted from the PolicyCenter configuration until you commit your changes.

Prior to committing your change list, you can click Cancel Deletion to restore the deleted item. The gray overlay is removed and you can once again edit the item.

 

 

 

To navigate to related pages

Use the links under Go to to display pages where you can specify additional coverage term properties. The available links vary depending on the term type of the coverage term.
