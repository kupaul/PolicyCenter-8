Coverage Term Package Contents Page

Manage the contents of the selected coverage term package.

Adding a New Coverage Term Package Contents Item

1. Click Add to add a new row to the Contents table. Note that the new row is added to the end of the list. If needed, scroll down to see the new row. Click a cell in the row to add or edit a value.

2. Specify appropriate values in the following fields:

 

Field

Description

Name

Name of the coverage term package contents item.

 

• Edit the text directly in the field using the current language. You can set the current language in the User Settings page.

• Click Translate to display the Display Key Values by Language dialog box, where you can enter or edit the text in any supported language.

 

 

Value Type

Type of value this package contents item holds, which enables code that accesses the value to handle the value appropriately.

• Count

• Days

• Hours

• Money

• Percent

• Each item in the package contents much have its own row in which its value is defined.

Value

Value assigned to this coverage term package contents item. Each item in the package contents much have its own row in which its value is defined.

Aggregation Model

How the coverage term package contents item is aggregated:

• Annual

• Each accident

• Each incident

• Each common cause

• Per claim

• Per item

• Per occurrence

• Per person

Values in this list are defined in the CovTermModelAgg typelist.

Restriction Model

How the coverage term is restricted, as defined in the CovTermModelRest typelist. A claim system can use this field.

Typical values include Accident, Bodily Injury, Employee dishonesty, Medical, and many others.

Common operations

To revert changes to coverage term package contents items

 

The changes you make in the current page are stored in your active change list. You can

• Cancel all changes you have made in the current page by clicking Revert.

• Selectively cancel changes by clicking Changes to display the Changes panel, then clicking to open the Changes page, then clicking Revert in the Revert column.

Note:  When you click Revert, Product Designer reloads the entire product model from the workspace.

 

 

 

To validate coverage term package contents items

Validate your changes by clicking Validate. Validation errors appear at the top of the page on a red background. The errors are marked with and appear on a red background. When no validation errors exist, clicking Validate displays the message No validation errors marked with on a green background.

Note:  When you click Validate, Product Designer reloads the entire product model from the workspace. Therefore, Product Designer can display unexpected values after validation if other users have committed changes to the same workspace.

 

 

To duplicate a coverage term package contents item

 

1. Select the row to duplicate by clicking anywhere in a row.

2. Click Duplicate.

3. Change one or more of the characteristics of the item to make it unique. Identical items cause validation errors.

Duplicated items are added to your active change list. They are not duplicated in the PolicyCenter configuration until you commit your changes.

 

 

To delete coverage term package contents items

 

1. Select the row to delete by clicking anywhere in the row. To delete multiple items at once, select multiple rows.

2. Click Delete.

Deleted values are added to your active change list. They are not deleted from the PolicyCenter configuration until you commit your changes.

 

 

To navigate to related pages

Use the links under Go to to display pages where you can specify additional coverage term package properties.
