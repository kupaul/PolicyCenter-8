Coverage Reinsurance Page

Specify a reinsurance coverage group for the selected coverage. For more information on reinsurance, see the Application Guide.

Selecting a Reinsurance Coverage Group

• From the Reinsurance Coverage Group list, select the reinsurance coverage group to apply to the coverage. For information on using Studio to configure coverage groups, see “Configuring Reinsurance Management” in the Configuration Guide.

Removing a Reinsurance Coverage Group

• Click in the Reinsurance Coverage Group field, and then backspace to remove the name.

Specifying a Reinsurance Coverage Group Script

You can define a reinsurance coverage group script that returns an RICoverageGroupType typekey to set the reinsurance coverage group. If the script returns null, then PolicyCenter uses the value you specified in the Reinsurance Coverage Group list.

• Click Expand to expand the Reinsurance Coverage Group Script section, and then enter your script.

Common operations

To revert changes to a reinsurance coverage group selection

 

The changes you make in the current page are stored in your active change list. You can

• Cancel all changes you have made in the current page by clicking Revert.

• Selectively cancel changes by clicking Changes to display the Changes panel, then clicking to open the Changes page, then clicking Revert in the Revert column.

Note:  When you click Revert, Product Designer reloads the entire product model from the workspace.

 

 

 

To validate a reinsurance coverage group selection

Validate your changes by clicking Validate. Validation errors appear at the top of the page on a red background. The errors are marked with and appear on a red background. When no validation errors exist, clicking Validate displays the message No validation errors marked with on a green background.

Note:  When you click Validate, Product Designer reloads the entire product model from the workspace. Therefore, Product Designer can display unexpected values after validation if other users have committed changes to the same workspace.

 

 

To navigate to related pages

Use the links under Go to to display pages where you can specify additional coverage properties.
