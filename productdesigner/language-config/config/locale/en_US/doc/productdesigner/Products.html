Products Page

Configure products on the Products page. The Products table shows each product that has been defined:

 

Column

Description
Icon

Icon that represents the product. Add icons for new products by using Studio. Instructions are provided in the Product Model Guide.

Sequence

Order in which the product appears in PolicyCenter, relative to other products.

Name

Name of the product. Click the name to open the Product home page where you can edit the details of the selected product.

Code

Unique identifier for the product. The product code is established when you add a new product and thereafter cannot be changed.

Other operations

• Sort – Click any column header to sort by that column. Click again to reverse the sort order.

• Change order – If the table is sorted by Sequence, drag a single row to change the order in which it is listed. Alternatively, edit the sequence number to establish a new order. Sequence determines the order in which items appear in PolicyCenter PCF objects when they are sorted by Priority.

• Edit – Click the blue item name to open a page where you can edit the item.

 

 

 

Adding a Product

Adding a product creates a new line of business in PolicyCenter, and enables users to create submissions based on the new line of business.

Note:  In most cases, adding a new line of business requires both Product Designer and Guidewire Studio. In Product Designer, you define the product model patterns, including the product, policy lines, question sets, audit schedules, and system tables. In Studio, a developer defines the backing data model, page configurations, and the code required to fully support the line.

1. Click Add.

2. In the Add Product dialog box, fill in the following required fields:

 

Field

Description

Code

Unique identifier for the product, which must begin with a letter and be composed only of letters, numbers, and underscores.

Name

Name of the product as it appears in PolicyCenter.

Default Policy Term

Default contractual term of a policy within this product. PolicyCenter has predefined terms of 6 months, Annual, and Other. Other policy terms can be defined in Studio in the TermType typelist.

Policy

Name of one policy line to add to the product. Every product must have at least one policy line. If you are creating a package policy, you can add more policy lines later. If the needed policy line does not exist, select any policy line for now, and then correct the selection after you have added the needed policy line.

Product Type

Type of product: Personal or Commercial. This value enables PolicyCenter to show or hide portions of certain built-in user interface objects that are designed either for personal or commercial contexts.

Account Type

Types of accounts that can have a product of this type. Person, Company, or Any.

Available Date

The date on which the product becomes available. The default availability date is a user setting which is stored by Product Designer. It is set when you change this field. If a default availability date has not been set, then it defaults to the current date.

The default availability date is also set when you change the StartEffectiveDate on any of the availability pages.

3. Click OK to add the new product. The resulting Product home page enables you to edit your settings and define additional product details.

Deleting a Product

1. Select the product to delete by clicking anywhere in its row except the product name. To delete multiple products at once, use standard keyboard and mouse key combinations to select multiple rows. The selected rows are highlighted.

2. Click Delete.

The deleted product is added to your active change list. It is not deleted from the PolicyCenter configuration until you commit your changes.

Deleting a product deletes the entire line of business, and can have significant impact on other product model and data model objects throughout PolicyCenter.

WARNING  Deleting a product can have unanticipated and unintended consequences. Guidewire strongly recommends that you fully understand your product model configuration before taking this action. For example, if you do not correctly remove references to a deleted policy line from your product definition, PolicyCenter cannot synchronize the product model at server startup. As a consequence, it can be impossible to start the server thereafter, until the problem is corrected.

Common Operations

To revert changes to products

 

The changes you make in the current page are stored in your active change list. You can

• Cancel all changes you have made in the current page by clicking Revert.

• Selectively cancel changes by clicking Changes to display the Changes panel, then clicking to open the Changes page, then clicking Revert in the Revert column.

Note:  When you click Revert, Product Designer reloads the entire product model from the workspace.
