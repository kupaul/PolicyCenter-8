Product Modifier Availability Page

Manage availability for product modifiers. For more information on availability, see the Product Model Guide.

Adding a New Modifier Availability Rule

1. Click Add to add a new row to the Availability table. Note that the new row is added to the end of the list. If needed, scroll down to see the new row. Click a cell in the row to add or edit a value.

2. Specify appropriate dimensions in the following fields. Dimensions that are undefined are wildcards and can match against any value. If no rows match on any defined dimension, the modifier is unavailable.

 

Field

Description

StartEffectiveDate

Date on which the modifier value officially becomes available or unavailable, based on the setting of the Availability field.

If you change the StartEffectiveDate then this date becomes the default availability date when you create new product model patterns or add new availability rows.

EndEffectiveDate

Date on which the modifier value officially becomes unavailable or available, based on the setting of the Availability field.

Availability

Available – The modifier becomes available on the StartEffectiveDate and becomes unavailable on the EndEffectiveDate.

Unavailable – The modifier becomes unavailable on the StartEffectiveDate and becomes available on the EndEffectiveDate.

State

Jurisdiction to which this availability rule applies. Click Search to display the State (Jurisdiction) dialog box and select a jurisdiction.

UWCompanyCode

Underwriting company for which this modifier is available or unavailable, based on the setting of the Availability field. Click Search to display the UW Company Code dialog box and select an underwriting company.

JobType

Job type to which this availability rule applies. One use of this dimension is to make a modifier available or unavailable on a job-by-job basis.

Note:  PolicyCenter uses the row with the most specific availability rule to determine availability for the modifier.

Adding a Modifier Availability Script

 

If the needed availability rule cannot be specified by using the dimensions of the Availability table, you can use an availability script to specify a custom rule.

Note:  Although Product Designer enables you to write availability scripts, it does not validate or compile the scripts that you write, and does not provide real-time help with interactive feedback. Therefore, Guidewire recommends that you use Product Designer only to examine scripts, and use the Studio XML editor to write, verify, and compile scripts. For instructions for defining availability scripts in Studio, see “Configuring Availability” in the Product Model Guide.

To examine an availability script in Product Designer

1. Click Expand to expand the Availability Script section.

2. In the Availability Script field, view the Gosu script that defines the availability for the selected item. Availability scripts return true/false values. Because they are written in Gosu, they can more readily capture complex logic. However, availability scripts typically require more resources than availability tables.

PolicyCenter evaluates availability scripts after the availability lookup table determines that a selection is available. Therefore, an availability script can only limit the selections that are otherwise available.

 

 

Example

Following is an availability script for the Package Risk Discount modifier in the Commercial Package Policy:

return EffectiveDatedFields.Branch.Policy.PackageRisk != null

Adding Grandfather States

 

Grandfathering enables you to continue to offer a selection to existing customers after it is not otherwise available. Use the Add, Duplicate, and Delete links under Grandfather States to manage grandfathering.

 

1. Click Expand to expand the Grandfather States section.

2. Specify the appropriate dimensions in the following fields. Dimensions that are undefined are wildcards and can match against any value. If you leave all dimensions blank, grandfathering applies in all situations.

Field

Description

State

Jurisdiction in which this selection is grandfathered.

UWCompanyCode

Underwriting company that can offer grandfathering for this selection. Click Search to display the UW Company Code dialog box and select an underwriting company.

EndEffectiveDate

Last day on which grandfathering of this selection can be offered.

 

 

Common operations

To revert changes to modifier rate factor availability rules

 

The changes you make in the current page are stored in your active change list. You can

• Cancel all changes you have made in the current page by clicking Revert.

• Selectively cancel changes by clicking Changes to display the Changes panel, then clicking to open the Changes page, then clicking Revert in the Revert column.

Note:  When you click Revert, Product Designer reloads the entire product model from the workspace.

 

 

 

To validate modifier rate factor availability rules

Validate your changes by clicking Validate. Validation errors appear at the top of the page on a red background. The errors are marked with and appear on a red background. When no validation errors exist, clicking Validate displays the message No validation errors marked with on a green background.

Note:  When you click Validate, Product Designer reloads the entire product model from the workspace. Therefore, Product Designer can display unexpected values after validation if other users have committed changes to the same workspace.

 

 

To duplicate a modifier rate factor availability rule

 

1. Select the row to duplicate by clicking anywhere in a row.

2. Click Duplicate.

3. Change one or more of the characteristics of the item to make it unique. Identical items cause validation errors.

Duplicated items are added to your active change list. They are not duplicated in the PolicyCenter configuration until you commit your changes.

 

 

To delete modifier rate factor availability rules

 

1. Select the row to delete by clicking anywhere in the row. To delete multiple items at once, select multiple rows.

2. Click Delete.

Deleted values are added to your active change list. They are not deleted from the PolicyCenter configuration until you commit your changes.

 

 

To navigate to related pages

Use the links under Go to to display pages where you can specify additional product modifier rate factor properties.
