Question Set Home Page

Examine and edit the details of the selected question set. The page title includes the question set name followed by the question set code in brackets. For example:

PA Pre-Qualification [PAPersonalAutoPreQual]

Editing Question Set Settings

 

Name

Name of the question set, initially set in the Add Question Set dialog box. You can:

 

• Edit the text directly in the field using the current language. You can set the current language in the User Settings page.

• Click Translate to display the Display Key Values by Language dialog box, where you can enter or edit the text in any supported language.

 

 

Description

Description of the question set that can be viewed by other business analysts. You can:

 

• Edit the text directly in the field using the current language. You can set the current language in the User Settings page.

• Click Translate to display the Display Key Values by Language dialog box, where you can enter or edit the text in any supported language.

 

 

 

Question Set Type

Type of question set, initially set in the Add Question Set dialog box. Determines how the questions in the question set are used. In the base configuration, select one of the following types:

• PreQual – Determine whether the insured meets the basic requirements for coverage. In the base configuration, PolicyCenter display pre-qualification questions on the Qualification screen and uses pre-qualification question set exclusively in submission jobs.

• Product Qualification – Determine whether the applicant is qualified for the requested product.

• Location – Determine whether a particular location qualifies for the requested product. PolicyCenter can be configured to display these questions if the location fits a particular profile or has particular associated coverages. For example, “Is alcohol served at this location?” could be asked on businessowners products where the location has an industry code associated with restaurants. The answer could be associated with liquor liability coverage. For another example, “Does the location have a swimming pool?” could be asked if the location has an industry code associated with rental apartments or motels.

• Offering Selection – For lines that provide offerings, whether specific offerings are available to the applicant. Available offerings can be affected by the answers to these questions.

• Supplemental – Typically used to determine additional underwriting risk. For example, “Any employees under age 16?” could raise an underwriting issue in a workers’ compensation policy.

• Underwriting – Typically used to assess the quality of a risk. A higher-quality risk could qualify for automated approval and a favorable rating, while a lower-quality risk could result in a higher premium or disqualification. Underwriting questions typically are evaluated as a set rather than individually.

A developer can use Studio to define other question set types. For more information, see the Product Model Guide.

Answer Container Type

Answer container type that you selected when you added the question set. Established when you add a new question set and thereafter cannot be changed.

Common Operations

To cancel addition or revert changes to question set settings

 

The changes you make in the current page are stored in your active change list. If you have just added an item, click Cancel Addition to remove the newly-added item. If you are editing an existing item, cancel all changes you have made in the current page by clicking Revert. Or, selectively cancel changes by clicking Changes to display the Changes panel, then click Show Full Panel to open the Changes page.

 

 

 

To validate the question set

Validate your changes by clicking Validate. Validation errors appear at the top of the page on a red background. The errors are marked with and appear on a red background. When no validation errors exist, clicking Validate displays the message No validation errors marked with on a green background.

Note:  When you click Validate, Product Designer reloads the entire product model from the workspace. Therefore, Product Designer can display unexpected values after validation if other users have committed changes to the same workspace.

 

 

To delete the question set

While editing the question set settings in the Question Set home page, click Delete.

• If you have not yet committed the question set, Product Designer deletes the question set.

• If the question set has been committed, the portion of the page that was previously editable becomes gray and you cannot make further changes to it.

The deleted item is added to your active change list. It is not deleted from the PolicyCenter configuration until you commit your changes.

Prior to committing your change list, you can click Cancel Deletion to restore the deleted item. The gray overlay is removed and you can once again edit the item.

 

 

 

To navigate to related pages

Use the links under Go to to display pages where you can specify additional question set properties.
