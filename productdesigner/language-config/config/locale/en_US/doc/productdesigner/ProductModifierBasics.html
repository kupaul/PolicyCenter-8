Product Modifier Home Page

Examine and edit the details of the selected product modifier. The page title includes the product modifier name followed by the product modifier code in brackets. For example:

Package Risk Discount [CPPPackageDiscount]

Editing Basic Product Modifier Settings

 

Name

Name of the product modifier, initially set in the Add Modifier dialog box. You can:

 

• Edit the text directly in the field using the current language. You can set the current language in the User Settings page.

• Click Translate to display the Display Key Values by Language dialog box, where you can enter or edit the text in any supported language.

 

 

Description

Description of the modifier that can be viewed by other business analysts. You can:

 

• Edit the text directly in the field using the current language. You can set the current language in the User Settings page.

• Click Translate to display the Display Key Values by Language dialog box, where you can enter or edit the text in any supported language.

 

 

Typelist

Appears only for modifiers whose data type is typekey.

Typelist from which PolicyCenter displays modifier values that can be selected by a user.

Modifier Type

Displays the modifier type. For product modifiers, the only choice is ProductModifier.

Data Type

Data type of the product modifier as set in the Add Modifier dialog box. The data type is established when you add a new modifier and thereafter cannot be changed.

Schedule Rate

Appears only for modifiers whose data type is rate.

Indicates whether the rate modifier is a schedule rate modifier. This setting is established when you add a new modifier and thereafter cannot be changed.

Editing Product Modifier Display Properties

Click Expand to expand the Display section.

 

Field

Description

Display Justification

• Selected – Display the Justification field in PolicyCenter. If the user does not supply a justification, PolicyCenter displays an error.

• Cleared – Do not display the Justification field. No justification is needed to use the modifier.

Display Range

Appears only if the modifier data type is rate.

• Selected – Show the range of values in PolicyCenter for this modifier.

• Cleared – Do not show the range of values for this modifier.

Default Minimum

Appears only if the modifier data type is rate.

Minimum value that can be entered absent any jurisdiction-specific minimum value defined in the State Min/Max page. If you enter a lower value, PolicyCenter displays an error.

Default Maximum

Appears only if the modifier data type is rate.

Maximum value that can be entered absent any jurisdiction-specific maximum value defined in the State Min/Max page. If you enter a higher value, PolicyCenter displays an error.

Display Value Final

Appears only if the modifier data type is rate.

• Selected – Displays a field in PolicyCenter that enables the user to indicate whether the specified modifier value is a final value or an estimate at time of issuance.

• Cleared – Hides the Final field.

Manually Enabled

Appears only if modifier data type is rate and Schedule Rate is false.

• Selected – Does not automatically include the modifier on the policy. Enables discretion about whether the modifier is used and thus a distinction between the modifier having no value and having a value of zero.

• Cleared – Automatically includes the modifier on the policy.

Note:  The default minimum and maximum values set the overall possible range for this modifier if no jurisdiction-specific minimum and maximum values are defined. If the modifier type is Schedule Rate and rate factors are entered, each rate factor must have a value within its own minimum and maximum. In addition, the sum of all rate factors must be within the range defined by the overall modifier minimum and maximum values.

Common Operations

To cancel addition or revert changes to product modifier settings

 

The changes you make in the current page are stored in your active change list. If you have just added an item, click Cancel Addition to remove the newly-added item. If you are editing an existing item, cancel all changes you have made in the current page by clicking Revert. Or, selectively cancel changes by clicking Changes to display the Changes panel, then click Show Full Panel to open the Changes page.

 

 

 

To validate the product modifier

Validate your changes by clicking Validate. Validation errors appear at the top of the page on a red background. The errors are marked with and appear on a red background. When no validation errors exist, clicking Validate displays the message No validation errors marked with on a green background.

Note:  When you click Validate, Product Designer reloads the entire product model from the workspace. Therefore, Product Designer can display unexpected values after validation if other users have committed changes to the same workspace.

 

 

To delete the product modifier

While editing the product modifier settings in the Product Modifier home page, click Delete.

• If you have not yet committed the product modifier, Product Designer deletes the modifier.

• If the product modifier has been committed, the portion of the page that was previously editable becomes gray and you cannot make further changes to it.

The deleted item is added to your active change list. It is not deleted from the PolicyCenter configuration until you commit your changes.

Prior to committing your change list, you can click Cancel Deletion to restore the deleted item. The gray overlay is removed and you can once again edit the item.

 

 

 

To navigate to related pages

Use the links under Go to to display pages where you can specify additional product modifier properties. The available links vary depending on the data type of the product modifier.
