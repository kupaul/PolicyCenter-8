Coverage Home Page

Examine and edit the details of the selected coverage. The page title includes the coverage name followed by the coverage code in brackets. For example:

Contractual Liability - Railroads [GLContractualLiabRR]

Editing Basic Coverage Settings

 

Name

Name of the coverage, initially set in the Add Coverage dialog box. You can:

 

• Edit the text directly in the field using the current language. You can set the current language in the User Settings page.

• Click Translate to display the Display Key Values by Language dialog box, where you can enter or edit the text in any supported language.

 

 

Description

Description of the coverage that can be viewed by other business analysts.

 

• Edit the text directly in the field using the current language. You can set the current language in the User Settings page.

• Click Translate to display the Display Key Values by Language dialog box, where you can enter or edit the text in any supported language.

 

 

Covered Object

Coverable to which this coverage applies. Coverables are defined in Studio as part of the data model. Established when you add a new coverage and thereafter cannot be changed.

Existence

Specification of whether users are allowed to elect or decline the coverage. Select from these options:

• <defined by script> – Opens the Advanced section where you can specify an existence script that determines whether to add the coverage.

• Electable – Coverage can be added to the policy if needed.

• Required – Coverage is added to the policy and cannot be removed.

• Suggested – Coverage is added to the policy by default but can be removed if needed.

Blanket Group Type

In the base configuration, appears only if the Policy Line is Commercial Property.

Type of blanket to which this coverage can be added. One of:

• Direct Loss – Provides coverages for buildings.

• Time Element – Provides business income or extra expense coverage.

If you do not select a value, then you cannot add this coverage to a blanket.

Category

Coverage category assigned to this coverage. Coverage categories permit grouping and searching in PolicyCenter so you can more easily find specific coverages. The category does not affect the coverage in any other way. Therefore, you can change the category at any time. To define new coverage categories go to the Categories page.

Public ID of the coverage as used in a legacy policy system of record.

Covered Party Type

Exposure type that is used when integrated with a claim system. When integrated with ClaimCenter, this field corresponds to the LossPartyType typelist.

First Party – Protects or provides benefits to the insured against direct loss to covered property, such as a car, a building, or a type of physical property.

Third Party – Protects the insured against liability suits from other parties that have suffered a loss because of the insured’s actions.

Scripts section

 

Initialization Script

Gosu script that runs when the PolicyLinePattern creates a Coverage. The script has a PolicyLine symbol available that PolicyCenter correctly types based on the configuration information of the PolicyLinePattern. Use this script to perform actions when initializing the Coverage.

In the base configuration, Medical Payments [PAMedPayCov] in Personal Auto has an initialization script that enables an additional Limit coverage term option based on a combination of offering and jurisdiction.

 

Removal Script

Gosu script that runs when the PolicyLinePattern removes a Coverage. The script has a PolicyLine symbol that PolicyCenter correctly types based on the configuration information of the PolicyLinePattern. Use this script to perform actions when removing the Coverage.

In the base configuration, Contractors Tools - Scheduled [BOPToolsSchedCov] in the Businessowners line has an example of a removal script that removes scheduled items before removing the coverage.

 

Existence Script

Gosu script that runs when the Existence field is set to <defined by script>. The script has a PolicyLine symbol that PolicyCenter correctly types based on the configuration information of the PolicyLinePattern. The return value from the existence script must be a code from the ExistenceType typelist.

ExampleAssume that uninsured motorist coverage is required in Wisconsin but electable in Pennsylvania. Use a script to check the state and set existence for the coverage based on the state.

Common Operations

To cancel addition or revert changes to coverage settings

 

The changes you make in the current page are stored in your active change list. If you have just added an item, click Cancel Addition to remove the newly-added item. If you are editing an existing item, cancel all changes you have made in the current page by clicking Revert. Or, selectively cancel changes by clicking Changes to display the Changes panel, then click Show Full Panel to open the Changes page.

 

 

 

To validate the coverage

Validate your changes by clicking Validate. Validation errors appear at the top of the page on a red background. The errors are marked with and appear on a red background. When no validation errors exist, clicking Validate displays the message No validation errors marked with on a green background.

Note:  When you click Validate, Product Designer reloads the entire product model from the workspace. Therefore, Product Designer can display unexpected values after validation if other users have committed changes to the same workspace.

 

 

To delete the coverage

While editing the coverage settings in the Coverage home page, click Delete.

• If you have not yet committed the coverage, Product Designer deletes the coverage.

• If the coverage has been committed, the portion of the page that was previously editable becomes gray and you cannot make further changes to it.

The deleted item is added to your active change list. It is not deleted from the PolicyCenter configuration until you commit your changes.

Prior to committing your change list, you can click Cancel Deletion to restore the deleted item. The gray overlay is removed and you can once again edit the item.

 

 

 

To navigate to related pages

Use the links under Go to to display pages where you can specify additional coverage properties.
