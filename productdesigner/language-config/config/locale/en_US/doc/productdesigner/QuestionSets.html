Question Sets Page

Manage question sets.

 

Column

Description

Sequence

Order in which the question set appears in PolicyCenter, relative to other question sets.

Name

Question set name. Click to open the Question Set home page where you can edit the details of the selected question set.

Code

Unique identifier for the question set.

Answer Container Type

Answer container type that enables questions to pertain to specific PolicyCenter entities. In the base configuration, one of:

• PolicyLocation

• PolicyPeriod

• PolicyLine

A developer can use Studio to define other answer container types. For more information, see the Product Model Guide.

Other operations

• Sort – Click any column header to sort by that column. Click again to reverse the sort order.

• Change order – If the table is sorted by Sequence, drag a single row to change the order in which it is listed. Alternatively, edit the sequence number to establish a new order. Sequence determines the order in which items appear in PolicyCenter PCF objects when they are sorted by Priority.

• Edit – Click the blue item name to open a page where you can edit the item.

 

 

 

Adding a Question Set

1. Click Add.

2. In the Add Question Set dialog box, fill in the following required fields:

 

Field

Description

Code

Unique identifier for the question set, which must begin with a letter and be composed only of letters, numbers, and underscores.

Name

Name of the question set as it appears in PolicyCenter.

Question Set Type

Type of question set that determines how the questions in the question set are used. In the base configuration, select one of the following types:

• PreQual – Determine whether the insured meets the basic requirements for coverage. In the base configuration, PolicyCenter display pre-qualification questions on the Qualification screen and uses pre-qualification question set exclusively in submission jobs.

• Product Qualification – Determine whether the applicant is qualified for the requested product.

• Location – Determine whether a particular location qualifies for the requested product. PolicyCenter can be configured to display these questions if the location fits a particular profile or has particular associated coverages. For example, “Is alcohol served at this location?” could be asked on businessowners products where the location has an industry code associated with restaurants. The answer could be associated with liquor liability coverage. For another example, “Does the location have a swimming pool?” could be asked if the location has an industry code associated with rental apartments or motels.

• Offering Selection – For lines that provide offerings, whether specific offerings are available to the applicant. Available offerings can be affected by the answers to these questions.

• Supplemental – Typically used to determine additional underwriting risk. For example, “Any employees under age 16?” could raise an underwriting issue in a workers’ compensation policy.

• Underwriting – Typically used to assess the quality of a risk. A higher-quality risk could qualify for automated approval and a favorable rating, while a lower-quality risk could result in a higher premium or disqualification. Underwriting questions typically are evaluated as a set rather than individually.

A developer can use Studio to define other question set types. For more information, see the Product Model Guide.

Answer Container Type

Answer container type that associates questions with specific PolicyCenter entities. In the base configuration, select one of the following types:

• PolicyLocation – Question set pertains to policy locations. For example, workers’ compensation workplace safety assessment questions.

• PolicyPeriod – Question set pertains to the policy period. For example, personal auto pre-qualification questions whose answers during subsequent policy periods could affect the desirability of continuing to offer the policy to the customer.

• PolicyLine – Question set pertains to the policy line itself. For example, workers’ compensation supplemental questions related to specific risks, such as operating aircraft or leasing employees to other businesses.

A developer can use Studio to define other answer container types. For more information, see the Product Model Guide.

3. Click OK to add the new question set. The resulting Question Set home page enables you to edit your settings and define additional question set details.

Deleting Question Sets

You cannot delete a question set that is used by a product. To delete question sets:

1. For each product that uses the question set you want to delete, navigate to the Product home page and then click Question Sets. The list of question sets associated with the product appears. Select the question set you want to delete and then click Remove. The question set is no longer associated with the product, but has not yet been deleted.

2. Navigate back to the Question Sets page, and then select the question set to delete by clicking anywhere in its row except the question set name. To delete multiple question sets at once, use standard keyboard and mouse key combinations to select multiple rows. The selected rows are highlighted.

3. Click Delete.

The deleted question set is added to your active change list. It is not deleted from the PolicyCenter configuration until you commit your changes.
