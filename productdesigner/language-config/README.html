Introduction

The user interface strings and online help for Product Designer reside in a web application archive (WAR) file named productdesigner.war. When PolicyCenter is first installed, productdesigner.war is located in the productdesigner/war directory within the PolicyCenter installation directory. In a Guidewire language pack, the same directory structure is used, but is distributed in a ZIP file named productdesigner.zip. You must unzip the distribution file into a working directory. Maintaining the directory structure provided by Guidewire, either in a PolicyCenter installation or in a language pack, is important to successful completion of the localization steps provided in this document.

Multiple user interface and online help language/locale combinations can be added to a single productdesigner.war file, enabling users to select their preferred settings while using Product Designer. The base configuration of Product Designer includes only U.S. English (en_US).

Note

If you do not want to localize Product Designer, skip this process. Deploy the U.S. English productdesigner.war as provided.

Localization Process Overview

The localization process involves the following steps, which are explained in detail in Detailed Localization Process, with matching step numbers:

1.       Run the provided script to unpack the user interface and online help language configuration files from productdesigner.war.

2.       For each language/locale you are adding, add a property in the language.properties file. If needed, change the default language property (en_US by default).

3.       Create a new language/locale directory for each language/locale you are adding and copy the contents of the en_US directory into it.

4.       Localize the content of the display.properties file to create a localized user interface.

5.       Localize the online help files.

6.       Run the provided script to repack the localized content into a new productdesigner.war file.

7.       Deploy the localized productdesigner.zip file to your Product Designer application server.

Prerequisites

Ensure that your computer meets the following requirements:

         Supported Java Development Kit (JDK) with jar.exe installed.

         JAVA_HOME/bin is in your PATH.

         The following directory structure contains the indicated files. This directory structure is part of the standard PolicyCenter installation. Alternatively, unzipping the Product Designer language pack creates this directory structure in your specified working directory.

productdesigner.war

Detailed Localization Process

This section provides detailed instructions for localizing the Product Designer.

To localize the Product Designer user interface

1.       Unpack the user interface and online help files from productdesigner.war. To do so, navigate to the productdesigner/language-config directory and run unpack.bat.

This command extracts from ../war/productdesigner.war the files needed for user interface localization, and puts them in the language-config/config directory. Running this command creates the following directories and files:

productdesigner

/language-config

/config

/locale

language.properties

/en_US

display.properties

/doc

/productdesigner

(online help files)

2.       Configure the new language/locale properties as follows:

a.       Open language-config/config/locale/language.properties in a text editor.

b.       Add a new definition for each language, using the following format:

cc_LL = Description

cc is the country code.

LL is the locale code.

Description is the description of the language and locale in the target language.

For example, to define French language for Canadian locale, add the following code and description:

fr_CA = Franais (Canada)

         The property name (fr_CA in the example) is the language/locale name and must match a language/locale directory name when the Product Designer application server starts. You will create this language/locale directory in Step 3.

         The property value (Franais (Canada) in the example) is the string that appears in the Language list on the User Settings page in Product Designer.

c.       To change the default language/locale, change the default.language property value to the language/locale code valid language/locale.

For example, to set the default language/locale to French/Canada, set the property as follows:

default.language = fr_CA

d.       Save and close language.properties.

3.       Create a new language/locale directory for each new language, as follows:

a.       Navigate to the language-config/config/locale directory.

b.       Create a new directory using the language/locale code you specified as a new property in the language.properties file in step 2b. For example, if you are adding French/Canada, create a new directory named:

/language-config/config/locale/fr_CA

c.       Repeat step b for each new language/locale added to language.properties.

d.       Copy the contents of an existing language/locale directory into the newly created language/locale directories. For example, copy the contents of en_US into fr_CA. The copied files will serve as a starting point for your localization. After you have completed this step, you should have a directory for each new language/locale. For example, if you are adding multiple languages:

language-config/config/locale

language.properties

/de_DE

display.properties

/doc

(help content)

/en_AU

display.properties

/doc

(help content)

/en_US

display.properties

/doc

(help content)

/fr_CA

display.properties

/doc

(help content)

/zh_CN

display.properties

/doc

(help content)

With this set of language/locale directories, your language.properties file must contain the following properties (with any one of the five language/locale codes as the default.language value):

de_DE = Deutsch (Deutschland)

en_AU = English (Australia)

en_US = English (United States)

fr_CA = Franais (Canada)

zh_CN = 中文

default.language = en_AU

Note

When you repack the WAR file in a later step, the set of languages/locales defined here comprise all the languages/locales that will be available in the resulting Product Designer user interface. Therefore, if you want to remove a language/locale from Product Designer, you can delete the corresponding language/locale directory and its corresponding property in the language.properties file. For example, if you do not want U.S. English available in Product Designer and delete the en_US directory. Also be sure to remove the en_US property from language.properties and make sure that the default.language property specifies a valid language/locale.

4.       Localize display.properties, as follows:

a.       Navigate to one of the language/locale directories you created in step 3b.

b.       Open display.properties in a text editor or localization tool.

c.       Localize each of the property values in the file, and then save and close display.properties.

5.       Localize the online help, as follows:

a.       Navigate to one of the language/locale directories you created in step 3b.

b.       Use a text editor or localization tool to localize the text of the each of the HTML files in the doc/productdesigner directory.

6.       Repack the localized content into a new productdesigner.war file. To do so, navigate to the productdesigner/language-config directory and run repack.bat.

This command does the following:

1)       Extracts all files from ../war/productdesigner.war to a temporary directory.

2)       Removes the old language configuration.

3)       Copies your new language configuration into the temporary directory.

4)       Repacks productdesigner.war in the language-config/output directory, so as not to overwrite the original productdesigner.war.

5)       Deletes the temporary directory.

Notes

         The Product Designer application server will not start if a language/locale is defined in language.properties but no corresponding language/locale directory exists.

         The Product Designer application server will not start if default.language is not defined or is not set to a valid language.

         There is no validation of the display.properties key for a given language. Therefore, you must take care to preserve and localize all needed display keys to preserve the integrity of the user interface.

7.       Deploy the localized productdesigner.zip file to your Product Designer application server, as follows:

Single-user Product Designer instance:

Copy productdesigner.war from the language-config/output directory to the original location (productdesigner/war) on each computer that runs Product Designer, overwriting the old productdesigner.war file in that location. Start Product Designer by following the instructions in the Product Designer Guide.

Multi-user Product Designer instance:

Copy productdesigner.war from language-config/output to your application server, and then deploy the WAR file by following the deployment instructions for your particular application server. Then start the application server and log in to Product Designer from a web browser using the appropriate URL.
